To add support for a new language, edit:

----------------------------------------
import newLanguage from "highlight.js/lib/languages/newlanguage";

Step 2: Register it in createConfiguredLowlight()
----------------------------------------
export function createConfiguredLowlight() {
  const lowlight = createLowlight();
  
  lowlight.register({
    javascript,
    typescript,
    // ... existing languages
    newlanguage,  // ← Add here
  });
  
  return lowlight;
}

Step 3: Add aliases (optional)
----------------------------------------
export const languageAliases = {
  // ... existing aliases
  "nl": "newlanguage",
  "new": "newlanguage",
};

Step 4: Add detection pattern (optional)
----------------------------------------
In autoDetectLanguage() function:
{ pattern: /^pattern\s+\w+/, language: "newlanguage" },

EXAMPLE: Adding Elixir support
-------------------------------
// Step 1
import elixir from "highlight.js/lib/languages/elixir";

// Step 2
lowlight.register({ ..., elixir });

// Step 3
export const languageAliases = {
  ...,
  "ex": "elixir",
  "exs": "elixir",
};

// Step 4
{ pattern: /^defmodule\s+\w+/, language: "elixir" },

/*
✅ GOOD PRACTICES:
  - Keep code blocks reasonably sized (< 100 lines recommended)
  - Don't use excessively long lines (< 120 chars preferred)
  - Syntax highlighting is cached by Highlight.js
  - CSS is optimized with !important flags for specificity

⚠️ THINGS TO AVOID:
  - Don't load hundreds of languages if only a few needed
  - Avoid duplicate style rules
  - Don't nest code blocks
  - Avoid mixed language detection within single block

📊 PERFORMANCE METRICS:
  - First highlight: ~5-50ms depending on code size
  - Cached highlight: <1ms
  - Theme CSS: ~3KB minified
  - Languages total: ~200KB (loaded on demand)
*/

// 9. TROUBLESHOOTING

/*
PROBLEM: Code not highlighting
SOLUTION: 
  - Check language is registered in setupHighlightLanguages.ts
  - Verify code block HTML structure is valid
  - Check browser DevTools > Elements tab for CSS classes
  - Try specifying language explicitly

PROBLEM: Wrong language detected
SOLUTION:
  - Explicitly set language in editor dropdown
  - Add better detection pattern to setupHighlightLanguages.ts
  - Check if code has ambiguous patterns

PROBLEM: Styles not applying
SOLUTION:
  - Verify syntaxHighlightingTheme import is correct
  - Check CSS is injected in <style> tag
  - Verify !important flags in CSS
  - Clear browser cache and hard refresh

PROBLEM: Performance issues
SOLUTION:
  - Check for too many code blocks on page
  - Verify no duplicate CSS rules
  - Use browser DevTools Performance tab
  - Consider lazy loading for long pages
*/
